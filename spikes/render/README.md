# MusicXML transformation and rendering spike

This spike treats MusicXML as the stored notation source. SVG and PDF are
derived outputs: they are regenerated from MusicXML and are never parsed to
recover notes.

It can:

- retain the full SATB score or extract soprano, alto, tenor, or bass;
- transpose the sounding pitches and key signature while preserving major or
  minor mode;
- change the displayed clef without changing any pitches;
- engrave every Verovio SVG page; and
- assemble those exact pages into a US Letter or A4 PDF.

The Open Hymnal MusicXML generated by `../ingest` has two staves and four
voices. `music21.voicesToParts()` returns those voices in S, A, T, B order. The
mapping is explicit in this spike and is rejected when fewer than four voices
are present.

## Run

From this directory, using an environment containing `music21`, `verovio`,
`CairoSVG`, and `pypdf`:

```bash
python -m hymn_render.cli \
  /path/to/hymn.musicxml \
  --output-dir build/o-for-a-thousand-tongues \
  --line bass \
  --target-key D \
  --clef bass \
  --page-size letter
```

`--line` accepts `satb`, `soprano`, `alto`, `tenor`, or `bass`. `--clef`
accepts `original`, `treble`, `bass`, `alto`, `tenor`, or `treble-8vb`.
A tonic-only `--target-key` inherits the source mode. An explicit destination
such as `D minor` must match the source mode; this is transposition, not
major/minor conversion.

The output directory contains:

- `score.musicxml`, the selected/transposed notation;
- `page-001.svg` and subsequent SVG pages;
- `score.pdf`, with one PDF page per SVG page; and
- `manifest.json`, including source/output hashes, pitch fingerprints, source
  and target keys, clef choice, tool versions, and render settings.

The pitch fingerprints make the semantic contract testable. A clef-only
render must have identical before/after pitch fingerprints. A transposition
must change pitches by the recorded interval; it is not implemented as a
display-only key-signature edit.
